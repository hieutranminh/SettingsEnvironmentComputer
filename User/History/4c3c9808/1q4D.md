# Vue 3 TypeScript Cursor Rules System

This directory contains a comprehensive set of cursor rules for a Vue 3 project using TypeScript, Vite, Pinia, Vue Router, PrimeVue v4, and Axios. These rules ensure consistent, maintainable, and scalable code across the entire project.

## 🏗️ Architecture Principles

### 1. Composition API First

- Always use `<script setup lang="ts">` for Vue components
- Prefer composables over mixins
- Use reactive references (`ref`, `reactive`) for state management

### 2. TypeScript Strict

- All code must be properly typed
- Use interfaces over type aliases for complex objects
- Avoid `any` type, use `unknown` or proper types
- Always define return types for functions

### 3. Functional Programming

- Prefer pure functions with clear input/output
- Avoid mutable state changes
- Use computed properties for derived state
- Minimize side effects

### 4. Modular Structure

- Organize code by feature/domain, not by type
- Use plugin-based architecture for extensibility
- Separate concerns with clear boundaries

## 🔧 Key Patterns

### Component Pattern

```vue
<template>
  <!-- Template content -->
</template>

<script setup lang="ts">
// Imports
import { ref, computed } from 'vue'
import type { ComponentProps } from '@/types'

// Props definition
interface Props {
  title: string
  items?: string[]
}

const props = defineProps<Props>()

// Emits definition
interface Emits {
  (e: 'update', value: string): void
}

const emit = defineEmits<Emits>()

// Reactive state
const loading = ref(false)
const data = ref<string[]>([])

// Computed properties
const filteredData = computed(() => {
  return data.value.filter((item) => item.includes(props.title))
})

// Methods
const handleUpdate = (value: string): void => {
  emit('update', value)
}
</script>
```

### Store Pattern

```typescript
import { ref, computed } from 'vue'
import { defineStore } from 'pinia'
import type { User } from '@/types'

export const useUserStore = defineStore('user', () => {
  // State
  const users = ref<User[]>([])
  const loading = ref(false)
  const error = ref<string | null>(null)

  // Computed
  const userCount = computed(() => users.value.length)

  // Actions
  const fetchUsers = async (): Promise<void> => {
    try {
      loading.value = true
      error.value = null
      users.value = await userService.getUsers()
    } catch (err) {
      error.value = err instanceof Error ? err.message : 'Failed to fetch users'
      throw err
    } finally {
      loading.value = false
    }
  }

  return {
    users,
    loading,
    error,
    userCount,
    fetchUsers,
  }
})
```

### API Service Pattern

```typescript
export class UserService {
  private api = useApi()

  async getUsers(params?: { page?: number; limit?: number }): Promise<User[]> {
    const queryParams = new URLSearchParams()
    if (params?.page) queryParams.append('page', params.page.toString())
    if (params?.limit) queryParams.append('limit', params.limit.toString())

    return this.api.get<User[]>(`/users?${queryParams.toString()}`)
  }

  async createUser(userData: CreateUserRequest): Promise<User> {
    return this.api.post<User>('/users', userData)
  }
}
```

## 🛡️ Error Handling

### Custom Error Classes

```typescript
export class ApiError extends Error {
  constructor(
    message: string,
    public statusCode: number,
    public endpoint: string,
  ) {
    super(message)
    this.name = 'ApiError'
  }
}

export class ValidationError extends Error {
  constructor(
    message: string,
    public field: string,
    public value: unknown,
  ) {
    super(message)
    this.name = 'ValidationError'
  }
}
```

### Error Handling Pattern

```typescript
const handleApiError = (error: unknown, context?: string): string => {
  if (error instanceof ApiError) {
    return getApiErrorMessage(error)
  } else if (error instanceof ValidationError) {
    return `Validation Error: ${error.message}`
  }

  return 'An unexpected error occurred'
}
```

## 📝 Code Style Rules

### Naming Conventions

- **Components**: PascalCase (e.g., `UserCard.vue`)
- **Composables**: camelCase with `use` prefix (e.g., `useUser`)
- **Stores**: camelCase with `use` prefix (e.g., `useUserStore`)
- **Services**: PascalCase with `Service` suffix (e.g., `UserService`)
- **Types**: PascalCase (e.g., `User`, `CreateUserRequest`)

### Import Organization

```typescript
// 1. Vue imports
import { ref, computed } from 'vue'

// 2. Third-party library imports
import { defineStore } from 'pinia'

// 3. Internal imports (with @ alias)
import type { User } from '@/types'
import { userService } from '@/services/userService'

// 4. Relative imports
import './styles.css'
```

### Error Handling Rules

1. **Never ignore errors** - Always handle or log them
2. **Use specific error types** - Create custom error classes
3. **Provide user-friendly messages** - Show clear, actionable errors
4. **Log technical details** - Include context for debugging
5. **Include recovery options** - Provide retry or reset functionality
