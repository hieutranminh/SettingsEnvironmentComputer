# AhaPlus Headquarters - Vue 3 Application

A modern Vue 3 application built with TypeScript, Pinia, Vue Router, and SCSS. This project follows best practices and includes a comprehensive set of essential features for building scalable web applications.

## 🚀 Features

- **Vue 3.5** with Composition API and `<script setup>`
- **TypeScript** for type safety
- **Pinia** for state management
- **Vue Router** with modular routing
- **Internationalization (i18n)** - Korean and English support
- **SCSS** with CSS custom properties and utility classes
- **Component Library** with reusable base components
- **Authentication System** with JWT tokens
- **API Client** with error handling and interceptors
- **Form Validation** utilities
- **Route Guards** for authentication and authorization
- **Responsive Design** with mobile-first approach

## 🛠️ Essential Components Generated

### 1. **Internationalization (i18n)**

- **Locales**: Korean (`ko.json`) and English (`en.json`) translation files
- **Composable**: `useI18n()` for translation and language management
- **Plugin**: Global i18n plugin for Vue application
- **Features**: Parameter interpolation, language switching, persistent storage

### 2. **TypeScript Types & Interfaces**

- **ApiResponse**: Generic API response types with pagination support
- **User**: User-related interfaces including roles and preferences
- **Router Types**: Extended Vue Router types with custom meta properties

### 3. **API Client & Services**

- **ApiClient**: HTTP client with authentication headers, error handling, and request/response interceptors
- **AuthService**: Complete authentication service with login, register, token management, and profile operations
- **Features**: Automatic token refresh, error handling, request cancellation

### 4. **State Management**

- **Auth Store**: Pinia store for authentication state with getters and actions
- **Composables**: Reusable Vue composition functions for auth and i18n
- **Features**: Reactive state, computed properties, async actions

### 5. **Routing System**

- **Modular Routes**: Organized route modules for different sections
- **Route Guards**: Authentication and role-based access control
- **Type Safety**: TypeScript support for route meta and navigation

### 6. **Component Library**

- **BaseButton**: Reusable button component with variants, sizes, and loading states
- **BaseInput**: Form input component with validation, error states, and accessibility
- **Features**: Consistent styling, accessibility support, TypeScript props

### 7. **Styling System**

- **CSS Variables**: Design system with colors, spacing, typography, and shadows
- **Utility Classes**: Responsive utility classes for common styling needs
- **Dark Theme**: CSS custom properties for theme switching
- **SCSS**: Modular SCSS with variables and mixins

### 8. **Form Validation**

- **Validation Utilities**: Field and form validation with custom rules
- **Email & Password**: Specialized validation for common fields
- **Debounce**: Utility for debouncing user input

### 9. **Constants & Configuration**

- **API Endpoints**: Centralized API endpoint configuration
- **App Constants**: Application-wide constants for validation, storage, and routes
- **HTTP Methods & Status**: Constants for HTTP operations

## 🚀 Getting Started

### Prerequisites

- Node.js 18+
- pnpm (recommended) or npm

### Installation

```bash
# Install dependencies
pnpm install

# Start development server
pnpm dev

# Build for production
pnpm build

# Run tests
pnpm test:unit

# Lint and format code
pnpm lint
pnpm format
```

### Environment Variables

Create a `.env` file in the root directory:

```env
VITE_API_BASE_URL=http://localhost:3000/api
```

## 📖 Usage Examples

### Using the Auth Store

```typescript
import { useAuthStore } from '@/stores/auth'

const authStore = useAuthStore()

// Login
await authStore.login({ email: 'user@example.com', password: 'password' })

// Check authentication
if (authStore.isAuthenticated) {
  console.log('User is logged in')
}

// Get user info
console.log(authStore.userFullName)
```

### Using Internationalization

```typescript
import { useI18n } from '@/composables/useI18n'

const { t, setLanguage, currentLanguage } = useI18n()

// Translate text
const message = t('common.loading')

// Change language
setLanguage('en')

// With parameters
const welcome = t('validation.minLength', { length: 8 })
```

### Using Base Components

```vue
<template>
  <BaseButton @click="handleClick" variant="primary" size="lg" :loading="isLoading">
    Click Me
  </BaseButton>

  <BaseInput v-model="email" label="Email" type="email" :error="emailError" required />
</template>
```

### Using API Client

```typescript
import apiClient from '@/services/api'

// GET request
const response = await apiClient.get<User>('/users/profile')

// POST request
const newUser = await apiClient.post<User>('/users', userData)

// PUT request
const updatedUser = await apiClient.put<User>('/users/profile', updates)
```
