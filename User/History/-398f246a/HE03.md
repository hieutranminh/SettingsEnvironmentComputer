# Global Dialog System

This project includes a comprehensive global dialog system built with PrimeVue v4 that supports nesting dialogs and customizable messages.

## Features

- ✅ **Nested Dialogs**: Support for multiple dialogs stacked on top of each other
- ✅ **Customizable Messages**: Full control over dialog content and styling
- ✅ **Type Safety**: Full TypeScript support with proper interfaces
- ✅ **Promise-based**: All dialogs return promises for easy async handling
- ✅ **Multiple Dialog Types**: Confirm, Alert, Error, Success, and Custom dialogs
- ✅ **PrimeVue Integration**: Uses PrimeVue's DialogService for consistent UI

## Quick Start

### 1. Import the Composable

```typescript
import { useGlobalDialog } from '@/composables/useDialog'
```

### 2. Use in Your Component

```vue
<script setup lang="ts">
import { useGlobalDialog } from '@/composables/useDialog'

const { showConfirm, showAlert, showError, showSuccess, showDialog } = useGlobalDialog()

const handleDelete = async (): Promise<void> => {
  const confirmed = await showConfirm('Are you sure you want to delete this item?')
  if (confirmed) {
    // Proceed with deletion
    await deleteItem()
    await showSuccess('Item deleted successfully!')
  }
}
</script>
```

## Available Methods

### `showConfirm(message: string, header?: string): Promise<boolean>`

Shows a confirmation dialog with Yes/No buttons.

```typescript
const confirmed = await showConfirm('Are you sure?', 'Confirm Action')
if (confirmed) {
  // User clicked "Yes"
} else {
  // User clicked "No" or closed the dialog
}
```

### `showAlert(message: string, header?: string): Promise<boolean>`

Shows an informational dialog with an OK button.

```typescript
await showAlert('Operation completed successfully!', 'Information')
```

### `showError(message: string, header?: string): Promise<boolean>`

Shows an error dialog with red styling and OK button.

```typescript
await showError('Something went wrong!', 'Error')
```

### `showSuccess(message: string, header?: string): Promise<boolean>`

Shows a success dialog with green styling and OK button.

```typescript
await showSuccess('Operation completed successfully!', 'Success')
```

### `showDialog(options: DialogOptions): Promise<boolean>`

Shows a fully customizable dialog with all available options.

```typescript
const result = await showDialog({
  header: 'Custom Dialog',
  message: 'This is a custom dialog with advanced options.',
  acceptLabel: 'Proceed',
  rejectLabel: 'Cancel',
  acceptClass: 'p-button-warning',
  rejectClass: 'p-button-secondary',
  acceptIcon: 'pi pi-check',
  rejectIcon: 'pi pi-times',
  draggable: true,
  resizable: true,
  maximizable: true,
  position: 'center',
})
```

## Dialog Options

The `DialogOptions` interface provides full control over dialog behavior:

```typescript
interface DialogOptions {
  header?: string // Dialog header text
  message?: string // Dialog message text
  acceptLabel?: string // Accept button text
  rejectLabel?: string // Reject button text
  acceptClass?: string // Accept button CSS class
  rejectClass?: string // Reject button CSS class
  acceptIcon?: string // Accept button icon
  rejectIcon?: string // Reject button icon
  closable?: boolean // Show close button
  draggable?: boolean // Allow dragging
  resizable?: boolean // Allow resizing
  modal?: boolean // Modal behavior
  position?: string // Dialog position
  maximizable?: boolean // Allow maximizing
  breakpoints?: Record<string, string> // Responsive breakpoints
  style?: Record<string, string> // Custom styles
  class?: string // Custom CSS class
  contentStyle?: Record<string, string> // Content styles
  contentClass?: string // Content CSS class
  maskStyle?: Record<string, string> // Mask styles
  maskClass?: string // Mask CSS class
  showHeader?: boolean // Show header
  baseZIndex?: number // Base z-index
  autoZIndex?: boolean // Auto z-index
  dismissableMask?: boolean // Close on mask click
  closeOnEscape?: boolean // Close on Escape key
  appendTo?: string // Append to element
  pt?: Record<string, unknown> // PrimeVue passthrough
  unstyled?: boolean // Unstyled mode
}
```

## Nested Dialogs Example

The system fully supports nested dialogs:

```typescript
const handleComplexFlow = async (): Promise<void> => {
  // First dialog
  const wantsToProceed = await showConfirm('Do you want to proceed?')

  if (wantsToProceed) {
    // Second dialog (nested)
    const confirmed = await showConfirm('Are you absolutely sure?')

    if (confirmed) {
      // Third dialog (nested)
      await showSuccess('Operation completed!')
    } else {
      await showAlert('Operation cancelled.')
    }
  }
}
```

## Error Handling

All dialogs return promises that resolve to `boolean`:

```typescript
try {
  const confirmed = await showConfirm('Proceed with action?')
  if (confirmed) {
    // Handle confirmation
  }
} catch (error) {
  // Handle any errors
  console.error('Dialog error:', error)
}
```

## Global Dialog Component

The `GlobalDialog` component is automatically included in `App.vue` and handles all dialog instances. No additional setup is required.

## Styling

The dialogs use PrimeVue's default styling and theme system. All dialogs automatically inherit your application's theme and styling.

## Best Practices

1. **Use descriptive messages**: Make dialog content clear and actionable
2. **Handle all cases**: Always handle both accept and reject scenarios
3. **Use appropriate dialog types**: Use `showConfirm` for user decisions, `showAlert` for information
4. **Keep dialogs simple**: Avoid complex logic in dialog callbacks
5. **Test nested scenarios**: Ensure your dialog flows work correctly when nested
