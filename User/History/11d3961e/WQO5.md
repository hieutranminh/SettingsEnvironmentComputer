# PDF Font Guide

This guide explains how to use custom fonts in PDF generation with jsPDF in the VueJS application.

## Overview

The application now supports custom font loading and management for PDF generation. The font system includes:

- **Font Utilities**: Core font management functions
- **Font Composable**: Vue composable for font management
- **Automatic Font Loading**: Fonts are automatically loaded when generating PDFs
- **Fallback System**: Graceful fallback to system fonts if custom fonts fail

## Available Fonts

### NanumGothic Font Family

- **NanumGothic Normal**: Regular weight for body text
- **NanumGothic Bold**: Bold weight for headers and emphasis

### System Fonts (Fallback)

- **Helvetica**: Default fallback font
- **Times**: Alternative fallback font

## Usage

### 1. Basic PDF Generation with Fonts

Fonts are automatically loaded when generating PDFs:

```typescript
import { generateReportPdf } from '@/workers/core/pdf-generator'

const pdfBlob = await generateReportPdf(options, data, config)
```

### 2. Using Font Composable

```typescript
import { usePdfFonts } from '@/composables/usePdfFonts'

const { availableFonts, isLoading, error, loadFonts, isFontAvailable, getRecommendedFonts } = usePdfFonts()

// Load fonts
await loadFonts()

// Check if font is available
const isAvailable = isFontAvailable('NanumGothic', 'bold')

// Get recommended fonts
const fonts = getRecommendedFonts()
```

### 3. Font Utilities (Advanced)

```typescript
import { addFontsToDocument, setFontWithFallback, getAvailableFonts } from '@/workers/utils/font-utils'

// Add fonts to document
addFontsToDocument(doc)

// Set font with fallback
setFontWithFallback(doc, 'NanumGothic', 'bold')

// Get available fonts
const fonts = getAvailableFonts(doc)
```

## Font Configuration

### FontConfig Interface

```typescript
interface FontConfig {
  name: string
  style: 'normal' | 'bold' | 'italic' | 'bolditalic'
  weight: 'normal' | 'bold'
  file: any // Font file content
}
```

### Adding New Fonts

1. **Add font files** to `src/workers/fonts/`
2. **Update font configuration** in `src/workers/utils/font-utils.ts`:

```typescript
export const AVAILABLE_FONTS: FontConfig[] = [
  // Existing fonts...
  {
    name: 'YourFont',
    style: 'normal',
    weight: 'normal',
    file: YourFontRegular,
  },
  {
    name: 'YourFont',
    style: 'bold',
    weight: 'bold',
    file: YourFontBold,
  },
]
```

3. **Update composable** in `src/composables/usePdfFonts.ts`:

```typescript
const getAvailableFonts = (): PdfFontInfo[] => {
  return [
    // Existing fonts...
    {
      name: 'YourFont',
      style: 'normal',
      weight: 'normal',
      isAvailable: true,
    },
    {
      name: 'YourFont',
      style: 'bold',
      weight: 'bold',
      isAvailable: true,
    },
  ]
}
```

## Best Practices

### 1. Font Selection

- Use **NanumGothic Bold** for headers and titles
- Use **NanumGothic Normal** for body text
- Use **Helvetica** as fallback for system compatibility

### 2. Error Handling

- Always use `setFontWithFallback()` instead of direct `setFont()`
- Check font availability before using custom fonts
- Provide meaningful fallback fonts

### 3. Performance

- Fonts are loaded once per PDF generation
- Font files are cached in memory
- Large font files may impact initial load time

### 4. Font File Requirements

- Font files must be in jsPDF-compatible format
- Use `.js` files generated by jsPDF font converter
- Ensure font files are properly imported

## Examples

### PDF Header with Custom Font

```typescript
import { setFontWithFallback } from '@/workers/utils/font-utils'

export const addPdfHeader = (doc: jsPDF, options: PrintPreviewOptions): void => {
  const pageWidth = doc.internal.pageSize.width

  // Title with bold font
  doc.setFontSize(28)
  setFontWithFallback(doc, 'NanumGothic', 'bold')
  doc.text(options.headerTitle || 'BÁO CÁO', pageWidth / 2, 10, { align: 'center' })

  // Subtitle with normal font
  doc.setFontSize(14)
  setFontWithFallback(doc, 'NanumGothic', 'normal')
  doc.text(options.headerSubtitle || '', pageWidth / 2, 15, { align: 'center' })
}
```

## Troubleshooting

### Common Issues

1. **Font not loading**

   - Check font file path and import
   - Verify font file format
   - Check console for error messages

2. **Font not displaying correctly**

   - Ensure font is properly added to document
   - Check font name and style parameters
   - Verify fallback font is working

3. **Performance issues**
   - Optimize font file size
   - Consider lazy loading for large fonts
   - Cache font instances when possible

### Debug Information

Enable debug logging to see font loading details:

```typescript
// Font loading logs
console.log('Font added successfully:', fontName)

// Font availability check
console.log('Available fonts:', getAvailableFonts(doc))

// Font usage logs
console.log('Using font:', fontName, style)
```

## Migration from Previous Version

If you were using fonts directly without the font utilities:

### Before

```typescript
doc.setFont('NanumGothic', 'bold')
```

### After

```typescript
setFontWithFallback(doc, 'NanumGothic', 'bold')
```

This ensures better error handling and fallback support.

## API Reference

### Font Utilities

- `addFontsToDocument(doc, fonts?)`: Add fonts to jsPDF document
- `setFontWithFallback(doc, fontName, style?, fallbackFont?)`: Set font with fallback
- `getAvailableFonts(doc)`: Get list of available fonts
- `isFontAvailable(doc, fontName, style?)`: Check font availability

### Font Composable

- `loadFonts()`: Load available fonts
- `getFontConfig(name, style?)`: Get font configuration
- `isFontAvailable(name, style?)`: Check if font is available
- `getRecommendedFonts()`: Get recommended fonts for different content types
