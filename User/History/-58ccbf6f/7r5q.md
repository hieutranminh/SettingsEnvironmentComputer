## 🔴 Vấn Đề Nghiêm Trọng

### 1. **CRITICAL: Async forEach Anti-pattern** ⚠️
**Location:** `bookings.js:308`, `bookings.js:283`

```javascript
// ❌ BAD - async/await trong forEach không hoạt động đúng
booking.bookedResources.forEach(async bookedResource => {
  await dispatch('setBookingResourceAvailable', {...})
  // ...
})
```

**Vấn đề:**
- `forEach` không đợi async callback hoàn thành
- Các dispatch chạy song song, không đảm bảo thứ tự
- Có thể gây race condition

**Giải pháp:**
```javascript
// ✅ GOOD - Sử dụng for...of hoặc Promise.all
for (const bookedResource of booking.bookedResources) {
  await dispatch('setBookingResourceAvailable', {...})
  // ...
}

// Hoặc nếu muốn parallel:
await Promise.all(booking.bookedResources.map(async bookedResource => {
  await dispatch('setBookingResourceAvailable', {...})
  // ...
}))
```

### 2. **CRITICAL: Logic lỗi trong anti-duplicate check**
**Location:** `bookings.js:319`

```javascript
// ❌ Logic sai - bookedResource là Object, không có .length
if (existedBooking?.bookingSource === BOOKING_SOURCE.NAVER || 
    (existedBooking && hasSingleResource) && bookedResource.length <= 1) {
  dispatch('removeBookingById', { bookingId: booking.bookingId })
}
```

**Vấn đề:**
- `bookedResource` là Object, không phải Array → `bookedResource.length` = undefined
- Operator precedence không rõ ràng (thiếu parentheses)
- Logic này không bao giờ chạy đúng cho case non-Naver

**Giải pháp:**
```javascript
// ✅ Fix logic
if (existedBooking?.bookingSource === BOOKING_SOURCE.NAVER || 
    (existedBooking && hasSingleResource && booking.bookedResources.length <= 1)) {
  await dispatch('removeBookingById', { bookingId: booking.bookingId })
}
```

### 3. **Race Condition trong addBooking**
**Location:** `bookings.js:303-330`

**Vấn đề:**
- Dòng 317: Đọc `existedBooking` từ bookingSet
- Dòng 304: Vừa mới set booking vào bookingSet
- Dòng 320: Remove booking vừa mới add → Không có tác dụng gì

**Flow hiện tại:**
```
1. setBookingSet(booking)          // Add to bookingSet
2. Check existedBooking            // Always true (vừa add ở bước 1)
3. removeBookingById               // Remove luôn cái vừa add
4. addOrUpdateItem                 // Add lại vào items
```

**Giải pháp:** Kiểm tra tồn tại TRƯỚC khi setBookingSet

## 🟡 Vấn Đề Trung Bình

### 4. **Memory Leak Risk: Dynamic Modules không được unregister**

```javascript
store.registerModule(['_calendar', 'bookings', bookingItemModuleName], cloneDeep(bookingItems))
```

**Vấn đề:**
- Modules được register khi có booking mới
- Không thấy logic unregister khi không còn booking
- Có thể gây memory leak khi calendar chạy lâu

**Giải pháp:** Cần thêm cleanup logic khi change date range

### 5. **Code Duplication giữa addBooking và parseBookingsToObj**

Cả 2 functions đều có logic:
- Convert timestamp
- Loop bookedResources
- Call setBookingResourceAvailable
- Call addOrUpdateItem

**Giải pháp:** Refactor thành shared helper function

### 6. **Missing Error Handling**

Nhiều async operations không có try-catch:
```javascript
booking.bookedResources.forEach(async bookedResource => {
  await dispatch('setBookingResourceAvailable', {...}) // No error handling

```
