# Custom ESLint Rules for Vue 3 + TypeScript

Bộ custom ESLint rules được thiết kế đặc biệt cho dự án Vue 3 + TypeScript, tuân thủ các coding standards và best practices của Ahasoft.

## 📋 Danh sách Rules

### 1. `enforce-handle-prefix`
Enforce việc sử dụng prefix "handle" cho các event handler functions.

**Tại sao:** Đảm bảo naming convention nhất quán cho event handlers, giúp code dễ đọc và maintain hơn.

**Ví dụ:**
```typescript
// ❌ Bad
const onClick = () => { console.log('clicked') }
const onKeyDown = (event: KeyboardEvent) => { console.log(event) }

// ✅ Good
const handleClick = () => { console.log('clicked') }
const handleKeyDown = (event: KeyboardEvent) => { console.log(event) }
```

**Cấu hình:**
```typescript
'custom/enforce-handle-prefix': ['error', {
  allowedPrefixes: ['handle'], // Mặc định: ['handle']
  functionNamePatterns: [/^on[A-Z]/, /^on[A-Z][a-z]/], // Mặc định: [/^on[A-Z]/, /^on[A-Z][a-z]/]
  checkArrowFunctions: true, // Mặc định: true
  checkFunctionDeclarations: true, // Mặc định: true
  checkFunctionExpressions: true, // Mặc định: true
}]
```

### 2. `enforce-vue-composition-api`
Enforce việc sử dụng Vue 3 Composition API và ngăn chặn Options API patterns.

**Tại sao:** Đảm bảo code sử dụng Composition API hiện đại thay vì Options API đã deprecated.

**Ví dụ:**
```vue
<!-- ❌ Bad -->
<template>
  <div>{{ message }}</div>
</template>

<script lang="ts">
export default {
  data() {
    return {
      message: 'Hello World'
    }
  },
  methods: {
    updateMessage() {
      this.message = 'Updated'
    }
  }
}
</script>

<!-- ✅ Good -->
<template>
  <div>{{ message }}</div>
</template>

<script setup lang="ts">
import { ref } from 'vue'

const message = ref('Hello World')
const updateMessage = () => {
  message.value = 'Updated'
}
</script>
```

**Cấu hình:**
```typescript
'custom/enforce-vue-composition-api': ['error', {
  allowOptionsApiIn: [], // Files được phép sử dụng Options API
  enforceScriptSetup: true, // Enforce <script setup> syntax
  checkDeprecatedPatterns: true, // Check Options API patterns
}]
```

### 3. `enforce-typescript-strict`
Enforce strict TypeScript patterns và ngăn chặn common anti-patterns.

**Tại sao:** Đảm bảo type safety và code quality cao.

**Ví dụ:**
```typescript
// ❌ Bad
function getUserData() { return userData }
const handleClick = () => { console.log('clicked') }
interface UserData { name: string }
import { UserData } from './types'

// ✅ Good
function getUserData(): UserData { return userData }
const handleClick = (): void => { console.log('clicked') }
interface IUserData { name: string }
import type { IUserData } from './types'
```

**Cấu hình:**
```typescript
'custom/enforce-typescript-strict': ['error', {
  enforceExplicitReturnTypes: true, // Enforce explicit return types
  preventAnyType: true, // Prevent 'any' type usage
  enforceInterfaceNaming: true, // Enforce interface naming (I prefix)
  enforceTypeImports: true, // Enforce type imports
  allowedAnyPatterns: [], // Allowed 'any' usage patterns
}]
```

## 🚀 Cài đặt và Sử dụng

### 1. Cài đặt Dependencies
```bash
pnpm install --save-dev @types/eslint @types/node
```

### 2. Import Custom Rules
```typescript
// eslint.config.ts
import { customRules } from './eslint-rules'

export default [
  // ... other configs
  {
    files: ['**/*.vue', '**/*.ts'],
    plugins: {
      'custom': {
        rules: customRules,
      },
    },
    rules: {
      'custom/enforce-handle-prefix': 'error',
      'custom/enforce-vue-composition-api': 'error',
      'custom/enforce-typescript-strict': 'error',
    },
  },
]
```

### 3. Chạy ESLint
```bash
# Lint tất cả files
pnpm lint

# Lint specific files
pnpm lint src/components/MyComponent.vue

# Fix auto-fixable issues
pnpm lint --fix
```

## 🧪 Testing

Chạy tests cho custom rules:

```bash
# Chạy tất cả tests
pnpm test

pnpm test --watch

pnpm test enforce-handle-prefix
```

## 🔧 Tạo Custom Rule Mới

### 1. Tạo Rule File
```typescript
// eslint-rules/my-custom-rule.ts
import type { Rule } from 'eslint'

const myCustomRule: Rule.RuleModule = {
  meta: {
    type: 'suggestion',
    docs: {
      description: 'My custom rule description',
      category: 'Best Practices',
      recommended: true,
    },
    fixable: 'code',
    schema: [],
    messages: {
      myMessage: 'My custom error message',
    },
  },

  create(context: Rule.RuleContext): Rule.RuleListener {
    return {
      // Your rule logic here
    }
  },
}

export default myCustomRule
```

### 2. Export Rule
```typescript
// eslint-rules/index.ts
import myCustomRule from './my-custom-rule'

export const customRules = {
  'my-custom-rule': myCustomRule,
  // ... other rules
}
```

### 3. Sử dụng Rule
```typescript
// eslint.config.ts
rules: {
  'custom/my-custom-rule': 'error',
}
```
