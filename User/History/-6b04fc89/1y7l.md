# Message Dialog Improvements

## Overview

The `useMessageDialog` composable has been significantly enhanced to provide a more flexible and extensible approach for displaying message dialogs. This document outlines the improvements and how to use the new features.

## Key Improvements

### 1. Enhanced Configuration System

#### Global Configuration

You can now configure default behaviors across your application:

```typescript
import { useMessageDialog } from '@/composables/useMessageDialog'

const { configure } = useMessageDialog()

// Configure global defaults
configure({
  queueDelay: 200,
  defaultStyle: {
    width: '500px',
    maxWidth: '95vw',
  },
  defaultBreakpoints: {
    '960px': '75vw',
    '641px': '90vw',
  },
  autoZIndex: true,
  baseZIndex: 1000,
})
```

#### Per-Dialog Configuration

Each dialog can have its own configuration that overrides global defaults:

```typescript
showInfo('Custom Dialog', 'Message content', {
  style: {
    width: '600px',
    height: '400px',
  },
  draggable: true,
  closable: false,
  dismissableMask: true,
})
```

### 2. Custom Actions Support

Dialogs can now have multiple custom action buttons:

```typescript
import type { DialogAction } from '@/composables/useMessageDialog'

const customActions: DialogAction[] = [
  {
    label: 'Save',
    severity: 'success',
    onClick: () => {
      // Handle save action
      console.log('Save clicked')
    },
  },
  {
    label: 'Cancel',
    severity: 'secondary',
    onClick: () => {
      // Handle cancel action
      console.log('Cancel clicked')
    },
  },
  {
    label: 'Delete',
    severity: 'error',
    onClick: () => {
      // Handle delete action
      console.log('Delete clicked')
    },
    disabled: false,
  },
]

show({
  type: 'info',
  title: 'Custom Actions',
  message: 'Choose an action:',
  actions: customActions,
})
```

### 3. Confirmation Dialogs

Built-in support for confirmation dialogs:

```typescript
showConfirm(
  'Confirm Action',
  'Are you sure you want to proceed?',
  () => {
    // User confirmed
    console.log('Confirmed!')
  },
  () => {
    // User cancelled
    console.log('Cancelled!')
  },
)
```

### 4. Event Emission Support

PrimeVue v4's DynamicDialog supports event emission. You can listen to events emitted by the dialog component:

```typescript
show({
  type: 'info',
  title: 'Event Example',
  message: 'This dialog will emit events when actions are clicked.',
  actions: customActions,
  emits: {
    onSave: (data?: unknown) => {
      console.log('Save event emitted:', data)
      // Handle save logic
    },
    onCancel: (data?: unknown) => {
      console.log('Cancel event emitted:', data)
      // Handle cancel logic
    },
    onDelete: (data?: unknown) => {
      console.log('Delete event emitted:', data)
      // Handle delete logic
    },
    onConfirm: (data?: unknown) => {
      console.log('Confirm event emitted:', data)
      // Handle confirm logic
    },
  },
})
```

The dialog component automatically emits events based on action button labels:

- Buttons with "Save" in the label emit `onSave`
- Buttons with "Cancel" in the label emit `onCancel`
- Buttons with "Delete" in the label emit `onDelete`
- Buttons with "Confirm" in the label emit `onConfirm`

### 5. Flexible Dialog Options

#### Styling Options

```typescript
showInfo('Styled Dialog', 'Content', {
  style: {
    width: '800px',
    height: '400px',
    maxWidth: '90vw',
    maxHeight: '80vh',
    minWidth: '300px',
    minHeight: '200px',
  },
})
```

#### Behavior Options

```typescript
showInfo('Interactive Dialog', 'Content', {
  draggable: true, // Allow dragging
  closable: false, // Hide close button
  dismissableMask: true, // Close on mask click
  closeOnEscape: true, // Close on Escape key
  modal: true, // Modal behavior
})
```

### 6. Queue Management

Enhanced queue management capabilities:

```typescript
const { getQueueLength, isQueueProcessing, clearQueue } = useMessageDialog()

// Check queue status
console.log(`Queue length: ${getQueueLength()}`)
console.log(`Processing: ${isQueueProcessing()}`)

// Clear all pending dialogs
clearQueue()
```

### 7. Immediate Display

Show dialogs immediately without queuing:

```typescript
const { showNow } = useMessageDialog()

// Show dialog immediately
await showNow({
  type: 'success',
  title: 'Immediate',
  message: 'This dialog shows immediately',
})
```

### 8. Generic Show Method

Maximum flexibility with the generic `show` method:

```typescript
const { show } = useMessageDialog()

show({
  type: 'info',
  title: 'Custom Dialog',
  message: 'Custom content',
  actions: customActions,
  style: customStyle,
  onClose: () => console.log('Closed'),
  draggable: true,
  closable: false,
  emits: {
    onSave: (data) => console.log('Save event:', data),
    onCancel: (data) => console.log('Cancel event:', data),
  },
  customProps: {
    // Any additional PrimeVue Dialog props
  },
})
```

## Type Safety

All new features are fully typed with TypeScript:

```typescript
interface DialogAction {
  label: string
  severity?: 'primary' | 'secondary' | 'success' | 'info' | 'warn' | 'error' | 'contrast'
  onClick: () => void
  disabled?: boolean
}

interface DialogStyle {
  width?: string
  height?: string
  maxWidth?: string
  maxHeight?: string
  minWidth?: string
  minHeight?: string
}

interface MessageConfig {
  type: 'success' | 'info' | 'warn' | 'error' | 'secondary' | 'contrast'
  title: string
  message: string | string[]
  onClose?: () => void
  actions?: DialogAction[]
  style?: DialogStyle
  closable?: boolean
  modal?: boolean
  draggable?: boolean
  dismissableMask?: boolean
  closeOnEscape?: boolean
  autoZIndex?: boolean
  baseZIndex?: number
  breakpoints?: Record<string, string>
  component?: Component
  customProps?: Record<string, unknown>
  emits?: Record<string, (data?: unknown) => void>
}
```

## Best Practices

### 1. Global Configuration

Set up global configuration early in your application:

```typescript
// In main.ts or App.vue
const { configure } = useMessageDialog()

configure({
  queueDelay: 200,
  defaultStyle: {
    width: '500px',
    maxWidth: '95vw',
  },
})
```

### 2. Custom Actions

Use custom actions for complex workflows:

```typescript
const handleDelete = () => {
  showConfirm('Delete Item', 'Are you sure you want to delete this item?', () => {
    // Perform delete
    deleteItem()
    showSuccess('Deleted!', 'Item has been deleted successfully.')
  })
}
```

### 3. Error Handling

Use the enhanced error dialog for better user experience:

```typescript
try {
  await performAction()
  showSuccess('Success!', 'Action completed successfully.')
} catch (error) {
  showError('Error', error.message, {
    onClose: () => {
      // Handle error dialog close
      console.log('Error dialog closed')
    },
  })
}
```

### 4. Queue Management

Monitor queue status for better UX:

```typescript
const handleMultipleActions = () => {
  showInfo('Processing', 'Starting multiple operations...')

  // Check if queue is processing
  if (isQueueProcessing()) {
    console.log('Dialog queue is currently processing')
  }

  // Clear queue if needed
  if (getQueueLength() > 5) {
    clearQueue()
    showWarning('Queue Cleared', 'Too many pending dialogs were cleared.')
  }
}
```

## Future Extensibility

The new structure is designed to be easily extensible for future needs:

1. **Custom Components**: Pass custom Vue components via the `component` property
2. **Animation Control**: Add animation configuration options
3. **Theming**: Support for theme-specific styling
4. **Accessibility**: Enhanced accessibility features
5. **Internationalization**: Better i18n support for action labels
6. **Analytics**: Built-in analytics tracking for dialog interactions

## Conclusion

The improved `useMessageDialog` composable provides a much more flexible and extensible foundation for displaying message dialogs. The new features enable better user experience, more complex workflows, and easier maintenance while maintaining backward compatibility with existing code.
